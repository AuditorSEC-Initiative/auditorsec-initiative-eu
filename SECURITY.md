# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| main    | Yes       |

## Reporting a Vulnerability

Do NOT open a public GitHub issue for security vulnerabilities.

### Contact
- **Email:** security@auditorsec.com
- **Response time:** Within 48 hours

### Process
1. Email with: description, steps to reproduce, impact assessment
2. Acknowledgment within 48 hours
3. Fix timeline provided
4. Credit in release notes (if desired)
5. Public disclosure after patch

## Scope
- Audit framework documents and compliance templates
- DAO governance smart contracts
- Integration APIs and authentication flows
- GDPR compliance mechanisms

## EU Compliance
This project adheres to EU NIS2 Directive and GDPR. Security incidents affecting EU critical infrastructure must be reported to authorities within 24 hours per NIS2 Article 23.

Thank you for helping keep AuditorSEC Initiative EU secure.
